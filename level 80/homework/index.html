TCP

TCP: Trannsmission Control Protocol

საკომუნიკაციო პროტოკოლი რომელიც პასუხს აგებს მონაცამების სანდოდ გადაცემაზე მონაცემებს ყოფს პაკეტებად და უზრუნველყოფს მათ სწორ გადაცემას გამოიყენება თითქმის ყველა სერვისში მაგალითად:საიტებზე, ელფოსტებზე,სერვერებზე

TLS

TLS: TRANSPORT LAYER SECURITY

უსაფრთხოების პროტოკოლი რომელიც მონაცემების უსაფრთხოებას უზრუნველყოფს ეს იცავს მონაცემების გადაცემას მესამე პირზე გამოიყენება HTTPS და სხვა ბევრ სერვისებში

HTTP

HTTP: HyperText TRANSFER PROTOCOL

პროტოკოლი რომელიც გამოიყენება ვებ სერვერსა და ბრაუზერს შორის უზრუნველყოფს საიტის ინფორმაციის გადაცემას და მონაცემები გადეცემა დასაშიფრად ასევე მუშაობს TCP ზე

HTTPS

HTTP: HyperText TRANSFER PROTOCOL SECURE

HTTP ის ბევრად უსაფრთხო ვერსიაა ის იყენებს TLS მონაცამების დასაშიფრად იცავს მომხმარებლის პირად ინფორმაციასა და მონაცემებს

HTTTP versions

HTTP/0.9(1991 წელი) მაგდროისათის ეგ ძალიან მარტივი იყო მას ქონდა მხოლოდ GET მეთოდი და იყებდა მხოლოდ HTML ტექსტურ მონაცემებსდა დოკუმენტების ჩატვირთის შემდეგ კი კავშირი იხურებოდა

HTTP/1.0(1996 წელი) ამ ვერსიაშ უკვე შეიძლებოდა სხვადასხვა ტიპის ფაილის გადაცემა (სურათები, ვიდეო, PDF და ა.შ.)მაგრამ: თითოეულ მოთხოვნაზე კავშირი იხსნებოდა და იხურებოდა და ნელა მუშაობდა

HTTP/1.1(1997 წელი) ყველაზე გრძელვადიანი ვერსია (რომელიც დღემდე ფართოდ გამოიყენება) TCP კავშირი შეიძლება გამოყენებულიყო და გაეკეთბინა პასუხი ბევრ მოთხოვნაზე ან პასუხზე დაემატა chunked transfer encoding სერვერს შეეძლო მონაცემის ნაწილ-ნაწილ გაგზავნა დაემატა ახალი მეთოდები: PUT, DELETE, OPTIONS
HTTP/2.0(2015 წელი) დაემატა Server Push სადაც სერვერს შეუძლია წინასწარ გაუგზავნოს რესურსები სანამ ბრაუზერი მოითხოვს ბევრად სწრაფია ვიდრე HTTP/1.1 განსაკუთრებით თანამედროვე ვებსაიტებისთვის.
HTTP/3(2022 წელი) ავტომატურად შიფრავს მონაცემებს (TLS 1.3) დახმარებით უკვე ფართოდ იყენებენ Google, Facebook, Cloudflare და სხვა დიდი კომპანიები. ბევრად სწარაფი და უსაფრთხოა ვიდრე წინა ვერსიები
